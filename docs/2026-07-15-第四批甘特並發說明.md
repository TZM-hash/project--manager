# 第四批：甘特、協作記錄與並發保護

## 升級內容

- 甘特工作新增里程碑、負責人、單一直接前置工作、實際開始日與實際完成日。
- 甘特圖以菱形顯示里程碑，並顯示負責人、前置工作及逾期標記；列印與 Excel 同步帶出強化資訊。
- 新增專案協作時間線，支援新增、修改、刪除、分頁、權限隔離及純文字正規化。
- `Projects`、`ProjectGanttPlans`、`ProjectCollaborationRecords` 使用 SQL Server `rowversion` 樂觀並發控制。
- 專案表單、進度表單、甘特與協作記錄會攜帶 Base64 版本值；過期提交不會覆蓋資料，畫面保留使用者輸入並提供重新載入入口。

## 甘特規則

- 里程碑的計畫開始／完成日必須相同；若填實際日期，實際開始／完成日也必須相同。
- 一般工作的完成日不可早於開始日，實際完成日不可早於實際開始日。
- 前置工作不可指向自己，也不可形成循環依賴。
- 未完成、計畫完成日早於今日且尚未填實際完成日時，判定為逾期。
- 負責人必須是該專案已分配人員；新工作第一次儲存取得 ID 後，可被其他工作選為前置工作。

## 權限與衝突處理

- 專案人員只能查看其被分配專案的協作記錄；管理員與主管可查看可存取的全部專案。
- 一般使用者只能修改或刪除自己建立的協作記錄；管理員與主管可管理全部記錄。
- 並發衝突回傳繁體提示，不自動合併或覆蓋；使用者重新載入最新版本後再決定是否重新送出。

## 資料庫與回復

- Migration：`EnhanceGanttCollaborationConcurrency`，只新增核准欄位、`ProjectCollaborationRecords` 資料表、外鍵與索引。
- 套用前建議先完成資料庫備份；回復程式碼時應同步執行 Migration Down，避免模型與資料庫不一致。
