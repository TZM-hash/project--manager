# Project Manager 第 1～2 批升级设计

## 1. 目标

本设计覆盖已确认升级路线中的前两个批次：

1. UI 基础与繁体术语统一。
2. 数据工作台与个人视图。

两批必须按顺序交付，每一批都能独立构建、测试、部署和回滚。第 1 批不改变数据库结构；第 2 批增加用户个人视图的持久化模型和 EF Core Migration。

## 2. 交付原则

- 以繁体中文作为源码、测试和产品文案的规范基准。
- 保留现有简体中文显示选项，但不再依赖混杂的简体源码作为转换输入。
- 优先改善高频数据页面的可读性和操作效率。
- 不改变现有角色名称、路由、Excel 格式和核心业务规则。
- 不在这两批引入即时聊天、后台任务中心、复杂审批流、甘特依赖关系或协作记录模型。
- 现有 `site.css` 和 `site.js` 不删除，改造成兼容入口，避免一次性破坏旧测试和部署缓存。

## 3. 第 1 批：UI 基础与繁体术语统一

### 3.1 可见成果

- 全站界面文案以繁体中文为规范来源，不再出现“当前、确定、推进、颜色、系统”等简体词混入繁体页面。
- 侧边栏按“业务工作／报表分析／系统管理”分区，并继续根据角色隐藏无权限入口。
- 折叠侧边栏后，所有图标菜单提供可见 Tooltip 和可访问名称。
- 数据页面降低阴影、发光和渐变强度，让表格内容成为视觉中心。
- 百分比进度条不再因为数值低就自动显示为危险色；危险、警告和成功色表达业务状态。
- 主要列表页使用统一空状态：说明没有数据的原因，并在有权限时提供下一步操作。
- 前端 JavaScript 和 CSS 按职责拆分，页面只初始化实际存在的组件。

### 3.2 繁体中文规范

#### 规范来源

Razor 页面、PageModel 错误消息、JavaScript 提示、测试断言和文档中的界面词，以繁体中文为规范文本。核心术语统一如下：

| 概念 | 规范词 |
| --- | --- |
| Project | 專案 |
| System | 系統 |
| User | 使用者 |
| Current | 目前 |
| Confirm | 確定 |
| Progress/Advance | 進度／推進 |
| Color | 顏色 |
| Data | 資料 |
| Import/Export | 匯入／匯出 |
| Print | 列印 |
| Settings | 設定 |
| Save | 儲存 |
| View | 檢視 |

新增一份机器可检查的术语规则，测试扫描界面源码中的已知混杂词。动态业务资料使用 `data-language-preserve` 保持原值，不自动改变客户名称、專案名稱、廠商名稱或人员姓名。

#### 简体显示

保留 `DisplayLanguageMiddleware` 和 `HtmlLanguageConverter`，但输入基准统一为繁体。简体模式继续转换标题、按钮、说明、Placeholder、Tooltip 和 ARIA 文案。转换器不得修改：

- `script`、`style`、`code`、`pre`、`textarea` 内容。
- 标记为 `data-language-preserve` 或 `translate="no"` 的内容。
- 下载文件、Excel 内容和数据库原始资料。

### 3.3 JavaScript 分层

`wwwroot/js/site.js` 变成轻量 ES Module 入口，仅负责：

1. 初始化全站 Shell。
2. 根据页面上的 `data-*` 标记按需动态载入组件模块。
3. 根据主题和特效设置按需载入视觉效果模块。

计划拆分为：

- `wwwroot/js/core/shell.js`：活动导航、侧边栏、导航分组、Tooltip 和通用键盘行为。
- `wwwroot/js/core/feedback.js`：按钮处理中状态、Toast 容器和通用反馈辅助函数。
- `wwwroot/js/components/bulk-actions.js`：批量选择和批量提交。
- `wwwroot/js/components/filter-drawer.js`：进阶筛选展开、关闭和键盘行为。
- `wwwroot/js/components/detail-tabs.js`：详情页签和 ARIA 状态。
- `wwwroot/js/components/rich-text.js`：富文本颜色工具。
- `wwwroot/js/components/gantt-editor.js`：甘特表单编辑器。
- `wwwroot/js/components/data-table.js`：列显示、行距、冻结列和表格设置事件。
- `wwwroot/js/pages/settings.js`：主题、动效和字体即时预览。
- `wwwroot/js/effects/ui-effects.js`：入场、数字滚动、卡片 Hover、页面转场和点击反馈。

每个模块导出单一 `init...()` 入口，并在重复调用时保持幂等。`site.js` 只有在页面存在对应选择器时才动态载入该模块；数据列表页不初始化字体预览，设置页不初始化甘特编辑器。

为兼容现有内联调用，`toggleSidebar` 和 `toggleNavGroup` 在第 1 批仍暴露到 `window`，同时逐步把 Razor 中的内联 `onclick` 改为 `data-*` 事件绑定。

### 3.4 CSS 四层结构

保留 `wwwroot/css/site.css` 作为兼容说明文件，实际页面按顺序载入四个层级：

1. `wwwroot/css/base.css`：设计 Token、字体、Reset、全局排版、Focus、Reduced Motion。
2. `wwwroot/css/components.css`：按钮、表单、卡片、表格、分页、徽章、Tooltip、Toast、空状态。
3. `wwwroot/css/pages.css`：首页、项目列表、详情、报表、月结、维保、甘特和登入页面结构。
4. `wwwroot/css/themes.css`：默认主题、清透主题、动效等级和字体变体。

打印规则集中放在 `pages.css` 的打印区，避免主题样式污染列印输出。拆分后不得通过复制选择器维持旧效果；每条规则只能有一个主要归属层。

### 3.5 导航信息架构

侧边栏改为三组：

- 业务工作：首页、我的專案、規劃中專案、專案總覽、專案分配、保養訂單、月結。
- 报表分析：未結案報表、歸檔歷史，以及后续统计入口。
- 系统管理：使用者、廠商、狀態、資料匯入匯出、系統設定。

组标题在没有任何可见子项时不渲染。折叠状态下组标题隐藏，菜单项通过 `data-nav-label` 显示 Tooltip；Tooltip 支持鼠标 Hover、键盘 Focus 和 Escape 关闭。侧边栏切换按钮提供 `aria-label` 和 `aria-expanded`。

### 3.6 业务语义颜色

项目状态颜色分为两类：

- 流程状态：依据 `ProjectStatus.Code`、`IsClosed` 和现有自定义样式显示成功、进行中、等待、阻塞或中性语义。
- 风险状态：依据收款落后、长期未更新、已结案等业务条件显示警告或危险。

第 1 批不新增状态数据库字段。新增纯代码的语义解析器，优先使用稳定状态 Code 和 `IsClosed`，未知自定义状态回退为中性样式，并保留管理员配置颜色作为小面积强调色。

百分比组件仅表达完成度：默认使用主色，100% 使用成功色。低百分比本身不是错误，不再自动使用红色。红色只用于阻塞、逾期、验证失败或破坏性操作。

### 3.7 空状态组件

新增共享局部视图和 ViewModel，至少支持：

- 标题。
- 原因说明。
- 可选主操作文字和链接。
- 可选清除筛选链接。
- 图标语义。

优先应用到專案總覽、我的專案、規劃中專案、未結案報表、月結、保養訂單、歸檔歷史和使用者管理。筛选后无结果时优先提供“清除筛选”；完全无资料且有新增权限时提供“新增”。

### 3.8 第 1 批测试与验收

- Release 构建通过。
- 现有 .NET 测试全部通过。
- JavaScript 所有模块通过语法检查。
- 术语扫描测试不允许已知简体词出现在规范界面源码中。
- 导航角色可见性不变。
- 折叠侧边栏可使用键盘访问 Tooltip。
- `prefers-reduced-motion` 下不运行持续动画。
- 最新视觉回归截图确认表格区域阴影更轻、文字对比清楚。

## 4. 第 2 批：数据工作台与个人视图

### 4.1 可见成果

- 未结案报表和主要业务表格默认只展示高优先级列，不再把全部字段压缩在一个视口内。
- 用户可切换系统预设视图，例如“进度管理、财务收款、请购追踪、完整资料”。
- 用户可保存自己的筛选、列显示、列顺序、行距和默认视图，并在不同浏览器或设备继续使用。
- 关键识别列在桌面横向滚动时保持冻结。
- 长说明使用截断和详情入口，不再撑高整行。
- 现有同步导入、导出和批量删除提供处理中状态与完成摘要；真正的后台任务进度中心留到第 6 批。

### 4.2 数据模型

新增 `SavedDataView`：

- `Id`：主键。
- `UserId`：Identity 使用者 ID。
- `PageKey`：稳定页面键，例如 `admin-projects`、`open-project-report`、`workbench-projects`、`maintenance-orders`。
- `Name`：使用者定义名称。
- `FilterJson`：经过白名单验证的筛选值。
- `ColumnJson`：列可见性和顺序。
- `RowDensity`：`Compact`、`Normal` 或 `Spacious`。
- `IsDefault`：是否为该页面默认视图。
- `CreatedAt`、`UpdatedAt`：审计时间。

约束：

- `(UserId, PageKey, Name)` 唯一。
- 每个 `(UserId, PageKey)` 最多一个默认视图，由服务层事务保证。
- 使用者删除时级联删除个人视图。
- JSON 长度受限，不保存任意 HTML、脚本或未经定义的字段。

该变更需要新增 EF Core Migration，并更新 SQL 数据字典、架构说明和维护手册。

### 4.3 服务边界

新增 `SavedDataViewService`，负责：

- 获取当前使用者某页面的视图。
- 建立、重新命名、更新、设为默认和删除个人视图。
- 对 `PageKey` 和筛选字段执行白名单验证。
- 规范化 JSON，忽略未知列和已移除字段。
- 保证同一页面只有一个默认视图。

每个支持页面注册自己的视图定义，包括：

- 可保存的筛选字段。
- 可配置列清单。
- 默认可见列。
- 固定列。
- 系统预设视图。

PageModel 只负责把当前查询参数交给服务，不直接解析任意 JSON。

### 4.4 系统预设与个人视图

系统预设视图以代码配置，不写入数据库，不能被用户删除：

- 专案总览：日常管理、财务收款、请购追踪、完整资料。
- 未结案报表：风险关注、进度管理、财务收款、完整资料。
- 我的專案：我的进度、近期更新、完整资料。
- 保養訂單：执行进度、维保范围、完整资料。

个人视图可从当前状态保存。选择视图后，服务器生成规范查询字符串并重新加载页面，确保链接可复制、浏览器前进后退有效。列配置和行距由服务器输出初始状态，前端修改时通过受保护的 POST Handler 保存。

未登录用户不保存视图；当前涉及的数据工作台页面均要求登录。

### 4.5 超宽表格规范

所有数据工作台使用统一列元数据：

- `data-column`：稳定列键。
- `data-column-label`：显示名称。
- `data-column-priority`：默认优先级。
- `data-column-fixed`：不可隐藏的识别列或操作列。
- `data-column-truncate`：长文字截断规则。

未结案报表默认显示：專案工號、專案名稱、專案人員、進度、收款比例、狀態、推進判斷、專案金額和最後更新時間。母案、请购、付款、结案日期、更新人员和完整说明放入其他预设或列管理。

桌面端可冻结选择框、专案工号和专案名称；操作列保持可见但不得覆盖前一列。窄屏继续使用横向滚动，本批不重做完整移动卡片视图。

长说明默认显示两行，提供可访问的“查看完整内容”入口。表格行点击不得覆盖复选框、按钮和链接的原有行为。

### 4.6 操作反馈范围

第 2 批为现有同步操作增加：

- 提交后按钮禁用和处理中状态。
- 批量删除完成后显示成功数、失败数和失败原因。
- Excel 匯入显示总行数、成功、跳过和错误摘要。
- Excel 匯出显示准备中状态，并在浏览器开始下载后恢复按钮。

本批不引入队列、SignalR 或后台工作表。超过单次 HTTP 请求合理时间的真正异步任务统一留到第 6 批“任务进度中心”。

### 4.7 第 2 批测试与验收

- 新增迁移可在空数据库和现有数据库升级成功。
- `SavedDataViewService` 覆盖新增、更新、默认唯一、删除、跨用户隔离和非法字段过滤。
- 不同角色只能读取和修改自己的视图。
- 应用视图后筛选结果与直接使用相同查询参数一致。
- 未结案报表在 1440px 宽度下关键列不逐字换行，页面无横向溢出。
- 列管理、行距、冻结列和预设视图支持键盘操作。
- 200% 缩放下工具栏可换行，表格仍可横向滚动，操作按钮可访问。
- Release 构建、完整 .NET 测试、视觉回归和 JavaScript 语法检查全部通过。

## 5. 依赖顺序

1. 先建立第 1 批的四层 CSS、模块化 JavaScript、繁体术语和共享组件。
2. 在第 1 批稳定资源结构上实现第 2 批的数据表格组件。
3. 新增个人视图模型、服务和迁移。
4. 先接入專案總覽和未結案报表，再接入我的專案和保養訂單。
5. 最后补操作反馈、视觉回归和文档。

第 2 批不得反向在旧 `site.js` 或 `site.css` 中增加新的大段实现。

## 6. 风险与控制

### 前端拆分回归

风险：旧测试和页面依赖单文件中的选择器或全局函数。

控制：保留兼容入口；先建立资产清单测试，再逐模块迁移；每迁移一个模块运行相关页面测试。

### 简繁转换误改业务资料

风险：运行时转换客户、专案或人员名称。

控制：动态资料统一使用语言保留标记；增加包含业务名称的转换回归测试。

### 保存视图包含过期字段

风险：后续页面调整导致历史 JSON 无法解析。

控制：服务按当前页面定义过滤未知字段并回退默认配置，不因单个字段失效导致整页错误。

### 冻结列遮挡

风险：横向滚动时固定列宽度不一致或覆盖内容。

控制：固定列使用明确宽度和 CSS 变量计算偏移；视觉测试验证普通、折叠侧栏和 200% 缩放。

### 数据库迁移

风险：生产数据库新增表失败或回滚时遗留资料。

控制：迁移前备份；Migration 仅新增个人视图表和索引，不修改现有业务表；回滚只删除个人视图资料，不影响项目资料。

## 7. 明确不在第 1～2 批实现

- 首页个人工作台。
- 项目评论、附件和 @提醒。
- 甘特里程碑、负责人、依赖关系和实际日期。
- 项目和甘特并发控制。
- 数据库自动备份、健康检查、异常日志和磁盘监控。
- 后台任务队列和实时进度中心。
- 完整移动端卡片式数据工作台。

这些内容继续按已确认的第 3～6 批顺序处理。

## 8. 完成定义

第 1 批完成：繁体规范、导航分组、Tooltip、语义颜色、空状态和前端资源分层全部上线，且不改变数据库。

第 2 批完成：主要数据页面具备系统预设、个人保存视图、优化后的宽表格和同步操作结果反馈；个人视图可跨浏览器保留，并通过迁移、权限和视觉验证。
