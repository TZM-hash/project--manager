# 第三批：個人工作台首頁

## 升級內容

- 首頁改為依登入者權限產生的個人工作台，只保留一個主結論與一個主操作。
- 增加「我的逾期、待處理、近期節點、長期未更新」四個工作佇列，每區最多顯示五筆並提供可執行的空狀態。
- 專案人員只會看到分配給自己的專案；管理員、主管與查詢人員可查看其可存取的全部專案。
- 工作台連結會帶入 `AnalysisType`，專案列表可直接套用對應的風險篩選並保留在分頁、篩選摘要與個人檢視中。
- 首頁在 720px 有效寬度下改為單欄佈局，主操作可使用鍵盤啟用，並納入主要按鈕顏色對比檢查。

## 指標定義

| 佇列 | 定義 |
| --- | --- |
| 我的逾期 | 未結案、甘特計畫完成日早於今日且專案進度未達 100% |
| 待處理 | 狀態為等待／阻塞，或收款比例落後專案進度超過 25 個百分點 |
| 近期節點 | 未來 14 天內到期且進度未達 100% 的甘特任務 |
| 長期未更新 | 未結案且超過 30 天未更新的專案 |

主結論依「逾期 → 待處理 → 近期節點 → 長期未更新」排序；全部為空時顯示目前沒有高優先風險。

## 驗證與維護

- 服務測試涵蓋角色資料隔離、各佇列定義、清單上限與主結論優先順序。
- 首頁煙霧測試與 UI 資產測試確認單一主結論、單一主操作及四個佇列。
- Playwright 規格包含鍵盤啟用、720px 版面與主操作對比度。
- 若需回復舊首頁，可回退本批 Git 提交；本批不包含資料庫結構變更。
